------

请基于当前任务卡、前端代码、后端接口文档、历史联调报告和回归报告，执行前后端复联调：

1. 复核上一轮遗留问题是否解决
2. 核对 API URL、Method、请求参数、响应字段、错误码
3. 核对 TypeScript 类型与后端 DTO 是否一致
4. 测试主流程是否跑通
5. 测试异常场景：400/401/403/404/500/超时/空数据
6. 输出 Bug 列表并标明责任归属：前端/后端/契约/设计/产品
7. 对前端问题给出修复代码
8. 对后端问题给出接口修改建议
9. 修复后复测
10. 给出最终结论：Accepted / Accepted with notes / Rejected

输出：

- 联调范围
- 遗留问题复核
- API契约核对
- Bug列表
- 修复内容
- 复测结果
- 是否允许合并
- 是否需要再次联调

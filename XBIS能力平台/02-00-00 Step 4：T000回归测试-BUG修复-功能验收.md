# ⚠️ 测试原则

- 不允许只检查当前新增功能，也必须检查受影响的旧功能
- 不允许只看代码，必须按用户操作路径验收
- 不允许跳过错误态、空态、加载态
- 不允许把“优化不改变行为”当成免测理由
- 如果发现 Bug，必须给出修复代码
- 修复后必须重新执行回归检查

------

# 2. 回归测试清单

请生成并执行回归测试清单。

## 2.1 主流程回归

检查：

- 页面是否正常打开
- 主流程是否可完成
- 数据是否正确展示
- 操作后状态是否正确变化

## 2.2 API / 数据回归

检查：

- API 是否正常调用
- 请求参数是否正确
- 响应字段是否正确映射
- 错误响应是否正确处理

## 2.3 UI / 响应式回归

检查：

- 桌面端 ≥1200px 是否正常
- 移动端核心流程是否可用
- 表格/卡片/弹窗是否错位
- 横向滚动/折叠/断点是否正常

## 2.4 状态回归

检查：

- loading 状态
- empty 状态
- error 状态
- disabled 状态
- success 状态

## 2.5 旧功能回归

检查：

- 是否影响已有入口
- 是否影响旧页面
- 是否影响原有数据展示
- 是否影响已有交互逻辑

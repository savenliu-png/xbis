# 🔍 检查维度（必须逐项检查）

------

## 1️⃣ API契约一致性

- URL 是否一致
- Method 是否正确（GET/POST）
- 参数是否完整（query/body/path）
- 是否存在多余参数

------

## 2️⃣ 返回结构匹配

- 字段名是否一致
- 类型是否匹配（string / number / array）
- 是否存在缺失字段
- 是否使用了未定义字段

------

## 3️⃣ 数据结构一致性

- 是否符合任务卡定义（ability/job/executor）
- 是否存在字段命名冲突
- 是否存在结构嵌套错误

------

## 4️⃣ 状态流一致性（非常关键）

检查：

- pending / running / success / failed 是否匹配
- 是否存在前端定义但后端没有的状态
- 状态转换是否合理

------

## 5️⃣ 错误处理

- 是否处理 API error
- 是否处理空数据（empty）
- 是否处理 loading
- 是否处理超时/失败

------

## 6️⃣ Business Services 层检查（必须）

- 是否通过 Service 层调用 API
- 是否绕过业务接入层（XBIS ❌）

------

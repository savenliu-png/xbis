## 一、关键依赖链（必须遵守）

### 第一层：基础依赖（P0）

```
T001(Design Tokens)
    │
    ▼
T002(基础组件库)
    │
    ▼
T003(业务组件库)
    │
    ▼
T004(页面模板)
```

**说明**：前端所有任务依赖此链，必须最先完成。预计 2 周。

---

### 第二层：数据基础（P0）

```
T005(ability 模型) ─┐
                    ├──→ T036(数据迁移)
T006(job 模型) ─────┤
                    │
T007(executor 模型)─┘
```

**说明**：三个数据模型互相独立，可并行开发。T036 依赖 T005 和 T006。

---

### 第三层：API 层（P0）

```
T005 ──→ T008(ability API)
            │
            ├──→ T011(能力列表) ──→ T012(能力详情) ──→ T013(能力测试)
            │                           │
            │                           ▼
            │                       T014(能力接入)
            │                           │
            │                           ▼
            │                       T017(任务创建) ←──────┐
            │                                               │
            └──→ T033(首页) ◄─────────────────────────────┘

T006 ──→ T009(job API)
            │
            ├──→ T015(任务列表) ──→ T016(任务详情)
            │       │
            │       ├──→ T018(任务状态轮询)
            │       │
            │       └──→ T029(管理端任务运营)
            │
            └──→ T034(callback 机制)

T007 ──→ T010(executor API)
            │
            └──→ T028(管理端执行器管理)
```

**说明**：
- T008, T009, T010 可并行开发
- 前端页面在 API 完成后才能开始
- T017（任务创建）依赖 T012（能力详情），因为创建任务需要先选择能力

---

### 第四层：前端页面（P0/P1）

```
用户端核心页面：
    T011(能力列表) → T012(能力详情) → T013(能力测试)
                                        │
                                        ▼
                                    T014(能力接入)

    T015(任务列表) → T016(任务详情)
        │
        ├──→ T018(任务状态轮询)
        │
        └──→ T017(任务创建) ←── T012(能力详情)

用户端增强页面：
    T011 + T015 → T033(首页)

    T003 + T004 → T019(计费概览) → T020(套餐购买) → T021(账单)

    T003 + T004 → T022(开发者中心)

    T003 + T004 → T023(个人中心)

管理端页面：
    T003 + T004 → T025(概览)

    T008 → T026(能力管理列表) → T027(能力编辑)

    T010 → T028(执行器管理)

    T009 → T029(任务运营)

    T003 + T004 → T030(用户管理)

    T003 + T004 → T031(套餐配置)
```

---

### 第五层：导航重构（P1，影响现网）

```
T011 + T015 + T019 + T022 + T023 ──→ T024(用户端导航重构)

T025 + T026 + T028 + T029 + T030 + T031 ──→ T032(管理端导航重构)
```

**说明**：导航重构必须等所有页面完成后进行，且影响现网，需灰度发布。

## 六、使用建议

1. **每日站会**：关注阻塞任务，确保前置任务按时完成
2. **周计划**：按并行组分配任务，最大化团队产出
3. **风险预警**：提前 2 天预警可能延期的阻塞任务
4. **资源调配**：当某条路径提前完成，调配人员支援关键路径

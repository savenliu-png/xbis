# XBIS Milestone Backlog（版本规划）

> 基于 Sprint Backlog 36 个任务
> 输出日期：2026-04-23
> 版本策略：MVP 快速上线 → 增强版完善体验 → 平台化全面运营

---

## 版本总览

| 版本 | 周期 | 目标 | 包含任务数 | 核心功能 |
|---|:---:|---|:---:|---|
| **Milestone 1** | Week 1-6 | MVP 上线，核心路径可用 | 14 个 | 能力浏览、任务提交、结果查看 |
| **Milestone 2** | Week 7-9 | 增强版，完整体验 | 18 个 | 能力测试、套餐购买、管理端运营 |

### 上线标准

#### 功能验收
- [ ] 能力中心可浏览所有已发布能力
- [ ] 能力详情页可查看 Schema 和文档
- [ ] 任务列表可查看所有任务
- [ ] 任务详情可查看执行进度和结果
- [ ] 任务创建可成功提交
- [ ] 异步任务状态自动刷新

#### 技术验收
- [ ] TypeScript 类型完整，无 `any`
- [ ] 使用 Design Tokens，无散落样式
- [ ] 页面状态机完整（loading/empty/error/idle）
- [ ] API 错误处理完整
- [ ] 首屏加载 < 2s

#### 兼容性验收
- [ ] 桌面端 ≥1200px 正常显示
- [ ] 用户端移动端核心功能可用

### 用户价值
- 新用户可以在 10 分钟内了解平台能力
- 用户可以提交第一个任务并查看结果
- 管理员可以查看任务执行情况

### 风险与应对
| 风险 | 可能性 | 影响 | 应对措施 |
|---|---|---|---|
| uiSchema 渲染器延迟 | 中 | 高 | 先支持基础字段类型，复杂类型后续迭代 |
| 数据迁移不一致 | 低 | 高 | 双写验证，每日比对数据量 |
| API 性能不达标 | 中 | 中 | 增加缓存，优化查询 |

## Milestone 3：平台化（Week 10）

### 目标
完成导航切换、系统配置、回调机制，系统稳定运行。

### 包含任务

| 任务编号 | 功能名称 | 所属端 | 优先级 | 工期 | 说明 |
|:---:|---|:---:|---|:---:|---|
| T024 | 用户端导航重构 | 前端 | P1 | 3 天 | 导航重构（影响现网） |
| T032 | 管理端导航重构 | 前端 | P1 | 3 天 | 导航重构（影响现网） |
| T034 | callback 机制 | 后端 | P1 | 3 天 | 回调机制 |
| T035 | 系统配置增强 | 后端 | P1 | 2 天 | 系统配置 |

**总计：4 个任务，约 1 周**

### 上线标准

#### 功能验收
- [ ] 新旧导航切换完成，旧路由保留 302 重定向
- [ ] 用户端 6 项菜单、管理端 9 项菜单正常显示
- [ ] 任务完成回调正常工作，支持重试
- [ ] 系统配置支持首页推荐、分类管理
- [ ] 历史数据迁移完成，双写停止

#### 技术验收
- [ ] 旧路由重定向 30 天内有效
- [ ] 回调机制支持重试（最多 3 次，间隔 5 分钟）
- [ ] 系统配置变更日志完整
- [ ] 全量功能测试通过

#### 性能验收
- [ ] 导航切换无闪烁，体验流畅
- [ ] 回调接口响应 < 200ms
- [ ] 系统配置热更新，无需重启

### 用户价值
- 用户可以使用全新的导航结构，快速找到功能
- 开发者可以通过回调机制实时获取任务结果
- 运营人员可以通过系统配置灵活调整平台行为

### 风险与应对
| 风险 | 可能性 | 影响 | 应对措施 |
|---|---|---|---|
| 导航重构用户流失 | 中 | 中 | 保留旧路由 30 天，增加引导提示 |
| 回调机制不稳定 | 低 | 高 | 增加重试机制，监控回调成功率 |
| 数据迁移后不一致 | 低 | 高 | 停止双写前进行全量数据比对 |

## 版本发布计划

### 发布节奏

```
Week 1-2:   基础层开发
Week 3:     API 层开发
Week 4:     能力平台开发
Week 5-6:   任务系统开发 ──→ Milestone 1 发布（内部测试）
Week 7:     用户端增强 + 管理端开发
Week 8-9:   管理端开发 ──→ Milestone 2 发布（灰度上线）
Week 10:    导航重构 + 上线准备 ──→ Milestone 3 发布（全量上线）
```

### 灰度策略

#### Milestone 1（内部测试）
- 部署到测试环境
- 内部账号访问新页面
- 验证功能完整性
- 修复阻塞 Bug

#### Milestone 2（白名单灰度）
- 生产环境部署，默认不可见
- 通过系统配置开启白名单
- 选择 10% 用户试用新页面
- 监控用户反馈和系统指标

#### Milestone 3（全量切换）
- 监控灰度用户反馈
- 修复问题后全量开启
- 保留旧页面入口 30 天
- 下线旧页面路由

| 维度 | Milestone 1 | Milestone 2 | Milestone 3 |

| **数据迁移** | 双写开始 | 双写持续 | 双写停止 |
| **回滚时间** | 随时 | 24 小时内 | 72 小时内 |

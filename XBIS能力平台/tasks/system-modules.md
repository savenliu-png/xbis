## 模块详细说明

### M1 基础设计系统（P0）

**目标**：建立统一的设计语言，确保用户端和管理端视觉一致性

**包含内容**：
- Design Tokens（颜色、字体、间距、圆角、阴影）
- 基础组件库（base/）：Button、Card、Input、Tag、Modal 等
- 业务组件库（business/）：AbilityCard、TaskItem、StatusBadge 等
- 页面模板（layout/）：ListPageShell、DetailPageShell、FormPageShell 等
- 页面块组件（blocks/）：AbilityGrid、TaskFilterBar、StatCards 等

**依赖**：无
**被依赖**：所有前端模块

---

### M2 后端数据层（P0）

**目标**：建立核心数据模型，支持能力平台和任务平台

**包含内容**：
- ability 表（能力定义，扩展自 api_market_item）
- ability_version 表（能力版本历史）
- ability_ui_schema 表（能力 UI 配置）
- executor 表（执行器定义）
- executor_health 表（执行器健康检查记录）
- job 表（任务，基于 invocation 扩展）
- job_result 表（任务结果）
- job_log 表（任务执行日志）
- job_stage 表（任务阶段进度）
- content_review 表（内容审核记录）

**依赖**：无
**被依赖**：M3 API 层

---

### M3 后端 API 层（P0）

**目标**：提供标准化的 RESTful API，连接数据层与前端

**包含内容**：
- `/api/v1/abilities/*` 能力相关接口
- `/api/v1/jobs/*` 任务相关接口
- `/api/v1/executors/*` 执行器相关接口
- `/api/v1/admin/*` 管理端接口
- Callback 机制（任务完成回调）
- 系统配置接口

**依赖**：M2 数据层
**被依赖**：M4-M17 所有前端模块

---

### M4 用户端 — 能力平台（P0）

**目标**：让用户发现、了解、测试、接入平台能力

**包含页面**：
- 首页（/home）：产品化展示，热门能力推荐
- 能力中心（/abilities）：能力网格、分类筛选、搜索
- 能力详情（/abilities/:id）：Schema 预览、任务提交入口
- 能力测试（/abilities/:id/test）：在线测试面板
- 能力接入（/abilities/:id/subscribe）：接入流程

**依赖**：M1, M3
**被依赖**：M5（任务创建需要选择能力）

---

### M5 用户端 — 任务系统（P0）

**目标**：让用户创建任务、监控执行、查看结果

**包含页面**：
- 任务列表（/jobs）：任务管理、筛选、批量操作
- 任务详情（/jobs/:id）：执行时间线、结果展示
- 任务创建（/jobs/create）：选择能力、填写参数

**依赖**：M1, M3, M4
**被依赖**：M9（管理端任务运营）

---

### M6 用户端 — 商业化（P1）

**目标**：支持套餐购买、账单管理、优惠券使用

**包含页面**：
- 计费概览（/billing）：当前计划、配额、用量
- 套餐购买（/billing/plans）：计划对比、支付
- 账单发票（/billing/invoices）：历史账单、发票申请
- 优惠券（/billing/coupons）：查看、激活

**依赖**：M1, M3
**被依赖**：M14（管理端套餐配置）

## 使用说明

1. **任务拆分时**：每个任务必须归属到一个模块
2. **排期时**：同一模块的任务可集中开发，减少上下文切换
3. **分工时**：按模块分配给不同开发者或小组
4. **验收时**：按模块逐一验收，确保模块内功能完整

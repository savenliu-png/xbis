## 二、AI 强制自检清单

### 2.1 设计规范检查

| 检查项 | 检查内容 | 通过标准 | 状态 |
|---|---|---|---|
| 任务独立性 | 每个任务是否可独立开发 | 有明确边界，不依赖其他任务内部实现 | ⬜ |
| 任务可测试性 | 每个任务是否可独立测试 | 有明确验收标准，可验证 | ⬜ |
| 任务可上线性 | 每个任务是否可独立上线 | 通过功能开关控制，不影响其他任务 | ⬜ |

## 三、开发后自检（Phase C 准入检查）

每个任务开发完成后，必须完成以下检查才能进入联调：

### 3.1 代码质量检查

- [ ] TypeScript 类型完整，无 `any`
- [ ] 使用 Design Tokens，无散落样式
- [ ] 使用页面模板骨架
- [ ] 页面状态机完整（loading/empty/error/idle）
- [ ] 组件复用符合分层规范（base → business → blocks）
- [ ] API 错误处理完整，用户可感知
- [ ] useEffect 清理完整，无内存泄漏
- [ ] 无 console.log 遗留（生产环境）

### 3.2 功能完整性检查

- [ ] 自测覆盖所有验收标准
- [ ] 空态处理（无数据时的展示）
- [ ] 加载态处理（加载中的展示）
- [ ] 错误态处理（请求失败的展示）
- [ ] 边界情况处理（超长内容、特殊字符、网络异常）
- [ ] 权限控制（无权限时的处理）

### 3.3 性能检查

- [ ] 首屏加载 < 2s
- [ ] 列表页支持虚拟滚动（>100 条数据时）
- [ ] 搜索防抖 300ms
- [ ] 图片懒加载
- [ ] 无内存泄漏（useEffect 清理）
- [ ] 大数据量表格响应 < 500ms

### 3.4 兼容性检查

- [ ] 桌面端 ≥1200px 正常显示
- [ ] 用户端移动端核心功能可用
- [ ] 管理端无需移动端适配
- [ ] 暗色模式正常显示（如适用）
- [ ] 主流浏览器兼容（Chrome/Firefox/Safari/Edge）

### 3.5 安全检奁

- [ ] 无敏感信息硬编码（API Key、密码等）
- [ ] XSS 防护（输入过滤、输出转义）
- [ ] CSRF 防护（请求头校验）
- [ ] 权限校验（前端 + 后端双重校验）

## 五、风险监控机制

### 5.1 每日监控

| 监控项 | 监控方式 | 阈值 | 响应措施 |
|---|---|---|---|
| 任务进度 | 每日站会 | 延期 > 1 天 | 预警，调整资源 |
| 阻塞任务 | 依赖关系图 | 前置任务延期 | 预警，启动并行方案 |
| Bug 数量 | 缺陷跟踪 | 新增 > 5 个/天 | 暂停新功能，集中修复 |

### 5.2 每周监控

| 监控项 | 监控方式 | 阈值 | 响应措施 |
|---|---|---|---|
| 里程碑进度 | 周会评审 | 完成率 < 80% | 调整范围或延期 |
| 风险状态 | 风险登记册 | 高风险项增加 | 升级处理，调整计划 |
| 代码质量 | 静态分析 | 警告 > 10 个 | 集中修复 |

### 5.3 每月监控

| 监控项 | 监控方式 | 阈值 | 响应措施 |
|---|---|---|---|
| 版本发布 | 发布评审 | 延期 > 1 周 | 调整范围或资源 |
| 用户反馈 | 反馈收集 | 负面反馈 > 20% | 紧急修复或回滚 |
| 系统稳定性 | 监控大盘 | 可用性 < 99.9% | 紧急修复 |

## 六、使用说明

1. **开发前**：完成 4.1 开发前检查
2. **开发中**：每日完成 4.2 开发中检查
3. **开发后**：完成 3.1-3.5 开发后自检 + 4.3 开发后检查
4. **上线前**：完成 4.4 上线前检查
5. **风险预警**：参考 5.1-5.3 风险监控机制

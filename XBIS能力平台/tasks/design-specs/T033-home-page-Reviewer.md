## 问题列表

### 问题1: 首页数据聚合 API 粒度过大
**严重程度: 中**

`GET /api/v1/home` 一次性返回所有区块数据，如果某个区块数据量大或加载慢，会阻塞整个页面渲染。

### 问题2: 未登录用户与已登录用户的差异化设计不足
**严重程度: 中**

设计方案仅提到"未登录用户展示简化版首页（无最近任务）"，但未说明其他区块的差异（如统计条是否展示真实数据、CTA 按钮文案差异等）。

### 问题3: 热门能力的排序算法未说明
**严重程度: 低**

PopularAbilities 展示"热门能力"，但未说明热门排序的依据（调用次数、订阅数、评分等）。

### 问题4: 缺少 SEO 相关设计
**严重程度: 低**

首页作为产品化入口，需要考虑 SEO（标题、描述、结构化数据等），但设计方案未提及。

## 修改建议

### 建议1: 考虑分块加载或骨架屏优化（建议修改）
- 方案A：保留聚合 API，但前端使用 Suspense 分块渲染
- 方案B：拆分为多个 API（`/api/v1/home/hero`, `/api/v1/home/stats` 等），并行加载

### 建议2: 明确未登录/已登录差异（必须修改）
```typescript
// 未登录用户
- HeroSection: CTA 为「免费试用」
- StatsBar: 展示平台总数据（能力总数/任务总数/用户总数）
- RecentJobs: 隐藏或展示示例任务

// 已登录用户
- HeroSection: CTA 为「创建任务」
- StatsBar: 展示个人数据（我的能力/我的任务/成功率）
- RecentJobs: 展示个人最近任务
```

### 建议3: 明确热门能力排序规则（建议修改）
推荐排序依据：
- 主排序：近 7 天调用次数（降序）
- 次排序：订阅用户数（降序）
- 兜底：创建时间（降序）

### 建议4: 增加 SEO 基础设计（建议修改）
- 页面标题：「XBIS 能力平台 - 发现、测试、调用 AI 能力」
- Meta Description：产品描述
- 结构化数据（Schema.org）：Organization、SoftwareApplication

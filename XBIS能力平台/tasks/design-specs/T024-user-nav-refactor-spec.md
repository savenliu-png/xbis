## 3. 数据流

```
[应用启动]
  └── 加载导航配置 ──► 渲染导航项

[用户点击导航]
  ├── 更新当前项高亮
  └── 跳转对应页面

[旧路由访问]
  ├── 匹配旧路由 ──► 302 重定向到新路由
  └── 保留旧路由 30 天
```

## 5. API 调用

无 API 调用。导航配置为静态配置。

---

## 6. 用户交互流程

### 主流程

```
用户访问页面
  ├── 加载导航 ──► 高亮当前项
  ├── 用户点击导航 ──► 跳转页面
  └── 用户点击移动端菜单 ──► 打开抽屉
```

### 异常流程

| 场景 | 触发条件 | 系统行为 | 用户反馈 |
|------|----------|----------|----------|
| 旧路由访问 | 访问旧路径 | 302 重定向 | 无感知 |
| 权限不足 | 访问无权限页面 | 跳转 403 | 提示无权限 |
| 页面不存在 | 访问无效路径 | 跳转 404 | 提示页面不存在 |

## 7. 边界与异常

| 场景 | 处理方案 |
|------|----------|
| 移动端适配 | 汉堡菜单 |
| 大量徽章 | 限制显示 99+ |
| 暗色模式 | 自动适配 |

---

## 8. 性能优化

- 导航渲染 < 100ms
- 无内存泄漏
- 支持懒加载

---

## 9. 风险点

| 风险 | 等级 | 说明 | 应对 |
|------|------|------|------|
| 旧路由兼容 | 高 | 需兼容旧路由 | 保留 30 天，302 重定向 |
| 用户体验变化 | 中 | 导航结构变化可能影响用户 | 灰度发布 |
| 移动端适配 | 中 | 移动端需调整布局 | 响应式测试 |

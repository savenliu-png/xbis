## 问题列表

### 问题1: 用户详情缺少权限管理功能
**严重程度: 中**

用户详情页未提及如何管理用户权限（如分配角色、调整权限范围）。

### 问题2: 限制配置的具体项未明确
**严重程度: 中**

LimitConfig 组件未说明可以配置哪些限制（如调用频率限制、配额限制、IP 白名单等）。

### 问题3: 用户搜索缺少高级筛选
**严重程度: 低**

FilterBar 只有用户名/邮箱搜索和状态/角色筛选，缺少注册时间、最后登录时间等高级筛选。

### 问题4: 用户禁用缺少原因记录
**严重程度: 中**

禁用用户时未要求填写禁用原因，不利于后续审计和追踪。

---

## 修改建议

### 建议1: 增加权限管理功能（建议修改）
用户详情页增加权限管理 Tab：
- 角色分配（管理员/用户/开发者）
- 能力访问权限
- API 调用权限

### 建议2: 明确限制配置项（必须修改）
```typescript
interface UserLimitConfig {
  maxCallsPerMinute: number;
  maxCallsPerDay: number;
  maxConcurrentJobs: number;
  allowedAbilities: string[];
  ipWhitelist?: string[];
}
```

### 建议3: 增加高级筛选（建议修改）
FilterBar 增加：
- 注册时间范围
- 最后登录时间范围
- 套餐类型筛选

### 建议4: 禁用用户时记录原因（必须修改）
禁用用户时弹出 Modal 要求填写：
- 禁用原因（必填）
- 禁用时长（永久/临时）
- 预计解禁时间（如临时禁用）

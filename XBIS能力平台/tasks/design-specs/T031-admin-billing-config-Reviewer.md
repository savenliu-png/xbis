## 问题列表

### 问题1: 计费规则编辑器复杂度过高
**严重程度: 高**

BillingRuleEditor 需要支持阶梯定价配置，这是一个高复杂度组件，设计方案未说明具体实现方式。

### 问题2: 套餐上下架缺少影响提示
**严重程度: 中**

上下架套餐会影响用户的购买和续费，但设计方案未说明是否需要提示影响范围。

### 问题3: 缺少套餐排序功能
**严重程度: 低**

套餐列表的展示顺序未说明，通常需要支持拖拽排序或手动设置排序权重。

### 问题4: 套餐预览功能未设计
**严重程度: 低**

保存套餐前，管理员可能需要预览套餐在用户端的展示效果。

---

## 修改建议

### 建议1: 简化 BillingRuleEditor 实现（必须修改）
采用表格形式配置阶梯定价：
```typescript
interface BillingTier {
  minUsage: number;
  maxUsage: number;
  unitPrice: number;
}

// 表格列：起始用量 | 结束用量 | 单价
// 自动校验区间连续性
```

### 建议2: 上下架时提示影响范围（建议修改）
上下架确认弹窗中显示：
- 当前订阅用户数
- 影响说明（"下架后新用户无法购买，已订阅用户不受影响"）

### 建议3: 增加套餐排序功能（建议修改）
支持拖拽排序或输入排序权重，控制套餐在前端的展示顺序。

### 建议4: 增加套餐预览功能（建议修改）
保存前提供「预览」按钮，在新窗口中查看套餐在用户端的展示效果。

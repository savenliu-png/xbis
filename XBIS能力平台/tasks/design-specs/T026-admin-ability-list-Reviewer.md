## 问题列表

### 问题1: 删除能力前未展示关联影响
**严重程度: 高**

删除能力会影响已订阅用户和进行中的任务，但设计方案仅在二次确认弹窗中显示能力名称，未展示关联影响（如订阅用户数、进行中任务数）。

### 问题2: 状态切换的乐观更新缺少回滚机制
**严重程度: 中**

设计方案提到"失败回滚"，但未详细说明回滚的实现方式（是本地状态回滚还是重新请求数据）。

### 问题3: 缺少批量操作功能
**严重程度: 中**

管理端列表通常需要批量操作（批量启用/禁用/删除），但设计方案未提及。

### 问题4: 表格列未明确是否支持排序
**严重程度: 低**

AbilityTable 未说明哪些列支持排序（如按名称、按创建时间、按状态排序）。

---

## 修改建议

### 建议1: 删除前展示关联影响（必须修改）
删除确认弹窗中增加：
- 当前订阅用户数
- 进行中任务数
- 历史任务总数
- 提示"删除后这些任务将无法执行"

### 建议2: 明确乐观更新回滚机制（必须修改）
```typescript
// 乐观更新流程
1. 前端先更新本地状态
2. 发送 PUT 请求
3. 请求成功：保持更新
4. 请求失败：恢复原始状态，显示错误提示
```

### 建议3: 增加批量操作功能（建议修改）
表格增加复选框列，支持：
- 批量启用/禁用
- 批量删除
- 全选/反选

### 建议4: 明确表格排序能力（建议修改）
支持按以下列排序：
- 名称（字母顺序）
- 创建时间（最新/最早）
- 状态（按状态优先级）
